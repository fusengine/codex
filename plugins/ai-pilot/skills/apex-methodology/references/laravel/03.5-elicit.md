## Laravel Priority Checks

### 1. Controller Architecture
```text
□ Single action controllers preferred
□ No business logic in controllers
□ Inject dependencies via constructor
□ Use Form Requests for validation
```

### 2. Eloquent Best Practices
```text
□ No N+1 queries (use with(), load())
□ Scopes for reusable queries
□ Accessors/Mutators for transformations
□ Soft deletes where appropriate
```

### 3. Service Layer
```text
□ Business logic in Services
□ Contracts/Interfaces in app/Contracts/
□ Dependency injection via Service Provider
□ Single responsibility per service
```

### 4. Security
```text
□ Form Request validation
□ Policy authorization
□ Mass assignment protection ($fillable)
□ SQL injection prevention (bindings)
```

## Example: Controller Review

```php
// Before elicitation
class UserController extends Controller
{
    public function store(Request $request)
    {
        $user = User::create($request->all());
        return response()->json($user);
    }
}

// After SEC-01 + SEC-02 + ARCH-01
class StoreUserController extends Controller
{
    public function __construct(
        private UserService $userService
    ) {}

    public function __invoke(StoreUserRequest $request): JsonResponse
    {
        $this->authorize('create', User::class);

        $user = $this->userService->create($request->validated());

        return response()->json($user, 201);
    }
}
```
