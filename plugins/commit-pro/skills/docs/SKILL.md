# Documentation Commit

## Preflight

Inspect documentation changes before acting:

```bash
git status
git diff --stat -- '*.md' '*.txt' '*.rst'
```

## Instructions

Create a documentation commit.

1. Stage only documentation files:
   - `*.md` files
   - `*.txt`, `*.rst` files
   - Files with only comment/JSDoc changes
2. Analyze files to determine scope
3. Generate commit message:
   - If `$ARGUMENTS` provided: `docs(scope): $ARGUMENTS`
   - If empty: generate from file names
4. Format: `docs(scope): imperative description`
5. **NO AI SIGNATURE**: Never add Co-authored-by, Generated by, or any AI mention
6. Ask confirmation before committing
7. Execute and show result
8. Execute the `post-commit` skill (CHANGELOG + version bump + git tag)
