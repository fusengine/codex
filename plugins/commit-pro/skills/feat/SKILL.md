# Quick Feature Commit

## Preflight

Inspect the current state before acting:

```bash
git status
git diff --stat
```

## Instructions

Create a quick feature commit.

1. Stage all changes: `git add -A`
2. Analyze modified files to determine scope
3. Generate commit message:
   - If the user provides context: `feat(scope): {user context}`
   - If empty: analyze diff and generate description
4. Format: `feat(scope): imperative description`
5. **NO AI SIGNATURE**: Never add Co-authored-by, Generated by, or any AI mention
6. Ask confirmation before committing
7. Execute and show result
8. Execute the `post-commit` skill (CHANGELOG + version bump + git tag)

**Note**: All commits trigger **PATCH** bump only (1.37.0 → 1.37.1). MINOR/MAJOR bumps are manual user decisions.
